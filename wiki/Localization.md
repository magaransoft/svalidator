SValidator offers localization through `ValidationResult`, `BindingResult`, and `BindingAndValidationResult`.

Provide a `Localizer` and call `localize` to translate message keys into localized strings. All formatting values are
preserved and applied after localization.

```scala
import com.magaran.svalidator.validation.Localizer

object MyLocalizer extends Localizer {
  def localize(input: String): String = input match {
    case "required.field" => "Field is required"
    case other => other
  }
}
```

Usage examples:

```scala
val validationResult = validator.validate(using instance).localize(using MyLocalizer)
val bindingResult = ObjectBinder.bind[Person](source, FieldKey.Root).localize(using MyLocalizer)
val bindingAndValidation = bindingValidator.bindAndValidate(source).localize(using MyLocalizer)
```

If you don't want localization, use `Localizer.NoLocalization`.

Localization in functional validators
=====================================

Functional validators handle localization differently. Instead of calling `.localize()` on the result, the `Localizer`
is provided as a given parameter to `withRules`, which applies it automatically before returning the `Either`.

```scala
given Localizer = MyLocalizer

val result = validator.validate(person, inputData)(using context)
// result is already localized — no need to call .localize()
```

Inside your validator, the `withRules` method picks up the `Localizer` from the calling scope:

```scala
class PersonFunctionalValidator
  extends FunctionalValidator[Person, RegistrationData, RequestContext, Unit] {

  override def validate(instance: Person, inputData: RegistrationData)(using ctx: RequestContext)
      : Either[Invalid, Unit] =
    WhenValidating(instance)
      .withSuccessData(())
      .withRules(  // accepts an implicit Localizer here
        For(_.firstName) ForField "firstName"
          must (_.nonEmpty) withMessage "required.field"
      )
}
```

If no `Localizer` is in scope, `Localizer.NoLocalization` is used by default.

The common pattern is to have your context type implement `Localizer`. Since the context is already a given parameter
in the `validate` method, `withRules` picks it up automatically — no extra wiring needed:

```scala
case class RequestContext(locale: Locale) extends Localizer {
  def localize(input: String): String = // look up translation for locale
}

class PersonFunctionalValidator
  extends FunctionalValidator[Person, RegistrationData, RequestContext, Unit] {

  override def validate(instance: Person, inputData: RegistrationData)(using ctx: RequestContext)
      : Either[Invalid, Unit] =
    WhenValidating(instance)
      .withSuccessData(())
      .withRules(  // ctx is used as the Localizer automatically
        For(_.firstName) ForField "firstName"
          must (_.nonEmpty) withMessage "required.field"
      )
}
```

This is the intended design for localization in functional validators.

Error message key conventions
=============================

SValidator uses string keys as message identifiers throughout validation and binding. These keys are passed to the
`Localizer` for translation, and any format values are applied afterward.

**Validation messages** are provided by you via `withMessage`. You can use literal text or keys for localization:

```scala
For(_.firstName) ForField "firstName"
  must (_.nonEmpty) withMessage "required.field"
  must (_.length <= 32) withMessage "max.length" withFormat 32
```

After localization, format values are substituted positionally using `{0}`, `{1}`, etc.

**Binding messages** are generated by `BindingLanguageConfig`. The default config uses these keys:

| Key | When used |
|-----|-----------|
| `"required.field"` | No value provided for a required field |
| `"invalid.value"` | Generic invalid value |
| `"invalid.nonempty.string"` | Whitespace-only string when style requires non-empty |
| `"invalid.boolean"` | Value cannot be parsed as boolean |
| `"invalid.integer"` | Value cannot be parsed as integer |
| `"invalid.long"` | Value cannot be parsed as long |
| `"invalid.float"` | Value cannot be parsed as float |
| `"invalid.double"` | Value cannot be parsed as double |
| `"invalid.decimal"` | Value cannot be parsed as BigDecimal |
| `"invalid.date"` | Value cannot be parsed as LocalDate |
| `"invalid.time"` | Value cannot be parsed as LocalTime |
| `"invalid.uuid"` | Value cannot be parsed as UUID |
| `"invalid.enumeration"` | Value does not match any enum id |
| `"invalid.sequence"` | Value cannot be parsed as a sequence |
| `"invalid.json"` | Value cannot be parsed as JSON |
| `"invalid.root.binding"` | Attempted to bind at root level for a type that requires a nested key |

You can customize these keys by implementing `BindingLanguageConfig` (see [Binding configuration](Binding-configuration.md)).
